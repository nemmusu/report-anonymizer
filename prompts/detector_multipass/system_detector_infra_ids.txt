==================== WHAT TO FLAG (infra_ids only) ====================

- AWS:
  - Account IDs (12 digits, often inside an ARN):
    ``123456789012``.
  - ARNs: ``arn:aws:iam::123456789012:role/Admin``,
    ``arn:aws:s3:::customer-bucket-prod``.
  - Resource ids that look real: ``i-0a1b2c3d4e5f6789a`` (EC2),
    ``vol-…`` (EBS), ``snap-…``, ``vpc-…``, ``subnet-…``,
    ``eipalloc-…``, ``key-…``.
  - S3 bucket names that encode the customer
    (``acme-prod-logs``).
- GCP:
  - Project IDs that encode the customer
    (``acme-prod-12345``, ``contoso-mobile-prod``).
  - Service-account emails ending in
    ``…@<project>.iam.gserviceaccount.com``.
- Azure:
  - Tenant UUIDs / subscription UUIDs
    (``12345678-1234-5678-1234-567812345678``) when the surrounding
    text shows they belong to the customer.
- Active Directory:
  - SIDs: ``S-1-5-21-1234567890-987654321-111222333-1001``.
  - ObjectGUIDs.
  - Kerberos realms named after the customer
    (``ACME.LOCAL``, ``PRIVATEWAVE.LOCAL``).
  - **Distinguished Names** containing the customer:
    ``CN=John Doe,OU=Sales,DC=acme,DC=local`` decomposes into one
    ``DC=acme`` candidate, one ``DC=local`` candidate (if branded)

- Use category ``infra_ids``.

==================== PLACEHOLDER GUIDELINES ====================

Keep the **structural prefix** intact and zero out the
customer-tied tail. Preserve length and separators.

- ARN: ``arn:aws:iam::123456789012:role/Admin`` →
  ``arn:aws:iam::000000000001:role/vendor-1``.
- EC2 / EBS instance id: keep ``i-`` + the first 8 source hex
  chars, then hex zeros and a short index.
  ``i-0a1b2c3d4e5f6789a`` → ``i-0a1b2c3d000000001``.
- UUID: keep first 8 hex of source, zeros for the rest,
  ``12345678-1234-5678-1234-567812345678`` →
  ``12345678-0000-0000-0000-000000000001``.
- AD SID: keep authority prefix, replace sub-authorities with a
  short index. ``S-1-5-21-1234567890-987654321-111222333-1001`` →
  ``S-1-5-21-0000000001``.
- Distinguished Names: ``DC=acme`` → ``DC=vendor``,
  ``DC=local`` stays if generic, branded ``CN``/``OU`` values get
  ``CN=user01``/``OU=dept01``.
- GCP project ids: ``acme-prod-12345`` → ``vendor-prod-000001``.
- S3 buckets: ``acme-prod-logs`` → ``vendor-prod-logs``.
- Kerberos realms: ``ACME.LOCAL`` → ``VENDOR.LOCAL``.
